## PART 1 — Launch EC2 Instance

### 1.1 — Create Instance
1. **AWS Console → EC2 → Launch Instance**
2. Settings:
   - **Name:** `flipkart-sentiment-app`
   - **AMI:** Ubuntu Server 22.04 LTS (Free Tier eligible ✓)
   - **Instance type:** `t2.medium` (4GB RAM — needed for MLflow + Prefect)
   - **Key pair:** Create new → name `flipkart-key` → download `.pem` file

### 1.2 — Security Group (Critical — open all 4 ports)

| Type | Protocol | Port Range | Source | Purpose |
|------|----------|-----------|--------|---------|
| SSH | TCP | 22 | My IP | SSH access |
| Custom TCP | TCP | **8501** | 0.0.0.0/0 | Streamlit App |
| Custom TCP | TCP | **5000** | 0.0.0.0/0 | MLflow UI |
| Custom TCP | TCP | **4200** | 0.0.0.0/0 | Prefect Dashboard |

3. **Storage:** 25 GB gp2
4. **Launch** → Go to EC2 Dashboard → note your **Public IPv4 address**
   - Example: `54.197.123.45` (replace this with your actual IP everywhere below)

## PART 8 — Start Prefect Workflow + Auto-Scheduling 🔄

You need **3 terminal sessions** (use `screen` for each):

```bash
# ── Session 1: Prefect Server ─────────────────────────────────────────────
screen -S prefect-server
source ~/flipkart-sentiment/venv/bin/activate
prefect server start
# Ctrl+A D to detach

# Wait 10 seconds for server to start, then:

# ── Session 2: Configure Prefect & Create Deployment ──────────────────────
screen -S prefect-deploy
source ~/flipkart-sentiment/venv/bin/activate
cd ~/flipkart-sentiment

# Point Prefect client to local server
export PREFECT_API_URL=http://127.0.0.1:4200/api

# Apply the deployment (creates daily schedule)
python prefect_deploy.py
# Ctrl+A D to detach

# ── Session 3: Start Worker Agent ─────────────────────────────────────────
screen -S prefect-agent
source ~/flipkart-sentiment/venv/bin/activate
export PREFECT_API_URL=http://127.0.0.1:4200/api
prefect agent start -q default
# Ctrl+A D to detach
```

### ✅ Access at: `http://54.197.123.45:4200`

### What you'll see in the Prefect Dashboard:

**Flows page:**
- `Flipkart Sentiment Analysis Pipeline`

**Deployments page:**
- `Daily Sentiment Pipeline`
- Schedule: `0 0 * * *` (every day at midnight IST)
- Parameters: `c_values: [0.1, 1.0, 5.0]`, `feature_sizes: [3000, 5000, 8000]`
- Tags: `sentiment`, `mlops`, `flipkart`

**Flow Runs page (after a run):**
- Status: ✅ Completed / ❌ Failed
- Duration, start time, run name
- Click a run → see individual task states

**Task breakdown inside each run:**

## PART 10 — Load Model from MLflow Registry in Python

```python
import mlflow.sklearn, mlflow

mlflow.set_tracking_uri('http://127.0.0.1:5000')   # or 'mlruns' for local
model = mlflow.sklearn.load_model(
    model_uri='models:/flipkart_sentiment_classifier/Production'
)

# Predict
texts = ['Amazing product, very durable!', 'Worst product, fake item.']
preds = model.predict(texts)
# [1, 0]  →  [Positive, Negative]
```

## PART 12 — Nginx Reverse Proxy (Optional Clean URLs)

```bash
sudo apt install nginx -y
sudo tee /etc/nginx/sites-available/flipkart > /dev/null << 'NGINX'
server {
    listen 80;
    server_name _;

    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
NGINX

sudo ln -sf /etc/nginx/sites-available/flipkart /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl restart nginx
```
Now access app at `http://54.197.123.45` (port 80, no port number needed)

## 🔧 Troubleshooting

| Problem | Solution |
|---------|----------|
| Can't reach `:8501` in browser | EC2 Security Group → add TCP inbound rule for port 8501, source 0.0.0.0/0 |
| Can't reach `:5000` (MLflow) | Same — add rule for port 5000 |
| Can't reach `:4200` (Prefect) | Same — add rule for port 4200 |
| `sentiment_model.pkl not found` | Run the full notebook first, then `scp` the file to EC2 |
| `NLTK LookupError` on first run | App auto-downloads; first request takes ~30s. Wait and refresh. |
| `ModuleNotFoundError` | Run `source venv/bin/activate` then `pip install -r requirements.txt` |
| MLflow UI shows empty | Ensure you're in the same folder that contains `mlruns/` when running `mlflow ui` |
| Prefect agent not picking up runs | `export PREFECT_API_URL=http://127.0.0.1:4200/api` before running agent |
| EC2 running out of RAM | Upgrade from `t2.micro` to `t2.medium` |
